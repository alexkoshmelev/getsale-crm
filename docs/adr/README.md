# Architecture Decision Records (ADR)

**Каноническое место** для ADR в этом репозитории — каталог **`docs/adr/`**.

## Формат файлов

- Имя: **`NNNN-kebab-case-title.md`**, где `NNNN` — порядковый номер (0001, 0002, …).
- Язык: по соглашению команды (RU/EN); для публичных интеграций предпочтительно EN в заголовке.
- Рекомендуемая структура внутри файла:
  1. **Статус** — proposed | accepted | superseded by ADR-XXXX
  2. **Контекст** — зачем решение
  3. **Решение** — что делаем
  4. **Последствия** — плюсы, минусы, миграции
  5. **Ссылки** — PR, `docs/`, internal API, код

## Связь с `ai_docs/`

Черновики, длинные приложения и материалы «для AI-аудита» могут жить в **`ai_docs/develop/architecture/`**, но **номерной ADR** и краткая фиксация решения — в **`docs/adr/`**. Из ADR давайте ссылку на вспомогательный документ, а не дублируйте полный текст в двух местах.
