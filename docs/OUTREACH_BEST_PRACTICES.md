# Лучшие практики: рассылки в Telegram (холодный аутрич)

Краткий чеклист и советы для эффективных и безопасных рассылок. Можно использовать в UI как подсказки или раздел «Перед запуском».

---

## Перед запуском кампании

1. **Прогрев аккаунтов** — перед массовыми рассылками желательно «прогреть» Telegram-аккаунт: регулярная активность, не только исходящие сообщения.
2. **Лимиты** — в системе включён дневной лимит сообщений на один BD-аккаунт (по умолчанию 40/день, настраивается через `CAMPAIGN_MAX_SENDS_PER_ACCOUNT_PER_DAY`). Соблюдайте задержку между отправками (sendDelaySeconds в кампании).
3. **Opt-out** — в первом сообщении добавлять возможность отказа от рассылки (например: «Если не актуально — напишите „стоп“»).
4. **Персонализация** — использовать переменные {{contact.first_name}}, {{company.name}} в шаблонах; избегать шаблонного спама. В редакторе шага кампании можно включить превью с тестовыми данными (Иван Тестов, ООО Пример), не выбирая контакт из CRM.
5. **Расписание** — отправлять в рабочие часы по таймзоне получателя; использовать настройки «Рабочие часы» и «Дни недели» в кампании.
6. **Один канал** — в v1 рассылки идут только в Telegram; соблюдать правила Telegram и не злоупотреблять массовыми сообщениями.

---

## Источники аудитории (GetSale CRM)

- **База CRM** — контакты с Telegram, фильтры по компании/воронке, возможность «только новые» (никогда не были в аутриче).
- **Файл CSV** — загрузка списка (telegram_id или email); система сопоставит или создаст контакты.
- **Группа в Telegram** — выбор из уже синхронизированных чатов (BD Accounts → выбор чатов при синхронизации); в аудиторию попадают контакты по сообщениям в выбранной группе.

Чтобы рассылать по участникам группы: сначала добавьте нужные чаты в BD Accounts (синхронизация), затем в кампании выберите источник «Группа в Telegram» и выберите чат.
